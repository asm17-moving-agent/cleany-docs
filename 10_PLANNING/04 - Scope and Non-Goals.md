# 범위와 제외 범위(Scope and Non-Goals)

## 1. 요약

이 문서는 끌리니 초기 기획에서 다룰 범위와 의도적으로 제외하거나 후순위로 둘 범위를 정리한다.

## 2. 현재 기획

- 장기 기획 범위는 무인 스터디카페/공간대여 시설의 이용 후 정리 자동화다.
- 7/10 회의의 1차 MVP 후보는 사전에 정한 소수의 쓰레기 물체를 인식·분류하고 집는 것이다.
- 분실물은 분류하되 집거나 이동하지 않으며, 공간 정돈·책상 닦기·소등·문단속은 1차 MVP에서 제외하는 방향으로 논의했다.

## 3. 왜 중요한가

범위가 넓어지면 로봇 하드웨어, 인식, 조작, 주행, 안전, 관제까지 모두 동시에 어려워진다. 초기 범위를 명확히 해야 데모 시나리오, Sprint 계획, 평가 기준을 현실적으로 정할 수 있다.

## 4. 핵심 내용

### 4.1 In Scope

| 항목                     | 설명                                         | 상태        |
| ---------------------- | ------------------------------------------ | --------- |
| 사전에 정한 쓰레기 물체 인식·분류·집기 | 소수 유형의 쓰레기 물체를 인식하고 지정 수거함으로 이동            | 1차 MVP 후보 |
| 분실물 후보 구분              | 쓰레기와 분실물을 구분하고 분실물은 건드리지 않음                | 1차 MVP 후보 |
| 자율주행                   | 대상 구역 이동 및 복귀. 현장 시연 여부는 지도·공간 안정성 검증 후 결정 | 시연 보강 후보  |
| 웹 대시보드                 | 운영자 호출 및 상태 확인의 최소 UI 후보                   | 범위 검토 필요  |
| 온디바이스 추론 검토            | 물체 탐지, 위치 추정, 장애물 회피 수행                    | 구현 대상     |

### 4.2 Non-Goals

아래 항목은 초기 문서화를 위한 가정 초안이며, 팀 검토 후 확정해야 한다.

| 항목                       | 제외 이유                                         | 상태           |
| ------------------------ | --------------------------------------------- | ------------ |
| 분실물 집기·보관                | 사용자 자산 처리 책임과 오분류 리스크가 있어 1차 MVP에서는 건드리지 않음   | 1차 MVP 제외 후보 |
| 공간 정리/정돈, 책상 닦기, 소등, 문단속 | 조작 대상과 안전 범위가 늘어나므로 후속 기능으로 둠                 | 1차 MVP 제외 후보 |
| 모든 무인점포 유형 완전 대응         | 업종별 공간 구조와 작업 대상이 다르므로 초기 범위가 과도해질 수 있음       | 1차 MVP 제외 후보 |
| 완전 자율 범용 로봇              | 소수의 정해진 물체와 시나리오에 우선 최적화                      | 1차 MVP 제외 후보 |
| 상용 수준 대규모 관제             | 대시보드는 최소 호출·상태 확인 외 요구사항이 정의되지 않음             | 1차 MVP 제외 후보 |
| 모든 자연어 명령 처리             | Rule-based VLA 중심이며 범용 자연어 명령 처리는 상세화되어 있지 않음 | 1차 MVP 제외 후보 |

### 4.3 범위 판단 기준 초안

- 이용 후 정리 시나리오와 직접 연결되는가?
- 사전에 정한 물체와 환경에서 안정적으로 시연할 수 있는가?
- XLeRobot 기반 로봇으로 데모 가능한가?
- 평가 기준을 정의할 수 있는가?
- 안전 리스크를 관리 가능한 수준으로 줄일 수 있는가?
