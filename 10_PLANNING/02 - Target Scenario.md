# 타깃 시나리오(Target Scenario)

## 1. 요약

기획서 기준 핵심 시나리오는 이용자가 퇴실한 뒤 로봇이 지정 공간으로 이동해 쓰레기, 분실물 후보, 흐트러진 집기류를 확인하고 기본 정리 작업을 수행해 다음 이용자가 사용할 수 있는 상태로 복구하는 것이다.

## 2. 현재 기획 이해

- 로봇은 사람이 상주하지 않는 시간에도 공간을 일정 수준 이상으로 유지하는 역할을 한다.
- 대상 공간은 무인 스터디카페 및 공간대여 시설을 우선 고려한다.
- 작업은 이용 종료 직후 또는 운영자가 지정한 시점에 수행되는 것으로 이해한다. 구체 트리거는 추가 확인 필요다.

## 3. 왜 중요한가

타깃 시나리오는 기능 범위와 기술 우선순위를 결정한다. 쓰레기 수거, 공간 정리, 자율주행, 책상 닦기, 소등, 문단속, 관제 대시보드가 모두 언급되어 있으나, MVP에서 어디까지 시연할지는 별도 합의가 필요하다.

## 4. 핵심 내용

### 4.1 기본 흐름 초안

1. 이용자가 공간 사용을 종료하고 퇴실한다.
2. 로봇이 지정된 작업 구역으로 이동한다.
3. 카메라와 RGB-D 정보를 활용해 책상, 바닥, 의자 주변의 물체 후보를 확인한다.
4. 물체를 쓰레기, 분실물 후보, 기타 물체로 분류한다.
5. 쓰레기 후보는 집기 가능 여부를 판단한 뒤 지정 수거함으로 이동한다.
6. 흐트러진 의자, 테이블 위 소형 집기, 사용 후 흐트러진 물체는 밀기, 정렬, 이동 동작으로 복구한다.
7. 책상 닦기, 소등, 문단속은 기획서에 포함되어 있으나 구체 수행 방식은 추가 확인 필요다.
8. 작업 완료 후 로봇은 대기 위치로 복귀한다.
9. 관제 대시보드는 상태 확인 또는 운영자 알림 위치로 포함될 수 있으나 MVP 포함 여부는 검토 필요다.

### 4.2 시나리오 참여자

| 참여자 | 역할 |
|---|---|
| 이용자 | 공간을 사용하고 퇴실하는 사람 |
| 운영자 | 무인 공간의 품질과 상태를 관리하는 사람 |
| 끌리니 로봇 | 이용 후 정리·점검 작업을 수행하는 로봇 |
| 관제 대시보드 | 상태 확인 및 기록 확인을 위한 placeholder 성격의 구성요소 |

### 4.3 작업 대상

- 쓰레기 후보
- 분실물 후보
- 의자 및 테이블 주변 정돈 상태
- 책상 표면 상태
- 조명 상태
- 출입문 잠금 상태

## 7. 미해결 질문

- 작업 시작 트리거는 예약, 센서 감지, 운영자 명령, 이용 종료 이벤트 중 무엇인가?

- 소등과 문단속은 실제 물리 조작인가, 상태 확인 및 알림인가?
- 책상 닦기는 MVP에 포함되는가?
- 실패 시 로봇은 대기, 재시도, 운영자 알림 중 어떤 정책을 따른다?
