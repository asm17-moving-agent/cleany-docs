# 타깃 시나리오(Target Scenario)

## 1. 요약

장기 시나리오는 이용 후 공간을 정리·점검하는 것이며, 1차 MVP 시나리오는 운영자가 호출한 뒤 로봇이 지정 구역에서 사전에 정한 쓰레기 물체를 분류하고 집는 흐름으로 좁힌다.

## 2. 현재 기획 이해

- 로봇은 사람이 상주하지 않는 시간에도 공간을 일정 수준 이상으로 유지하는 역할을 한다.
- 대상 공간은 무인 스터디카페 및 공간대여 시설을 우선 고려한다.
- 초기 작업은 자동 감지보다 운영자의 호출을 우선 검토한다. 호출 UI는 웹 대시보드 후보로 논의됐다.
- 1차 MVP에서는 쓰레기와 분실물을 구분하되, 분실물은 집거나 이동하지 않는다.

## 3. 왜 중요한가

타깃 시나리오는 기능 범위와 기술 우선순위를 결정한다. 쓰레기 수거, 공간 정리, 자율주행, 책상 닦기, 소등, 문단속, 관제 대시보드가 모두 언급되어 있으나, MVP에서 어디까지 시연할지는 별도 합의가 필요하다.

## 4. 핵심 내용

### 4.1 1차 MVP 흐름 초안

1. 운영자가 웹 대시보드 등으로 대상 구역의 작업을 요청한다.
2. 로봇이 대상 구역으로 이동한다. 현장 시연 가능 여부는 지도·공간 조건을 검증한 뒤 정한다.
3. 카메라와 RGB-D 등을 활용해 사전에 정한 물체 후보를 확인한다.
4. 물체를 쓰레기, 분실물 후보, 기타 물체로 구분한다.
5. 쓰레기로 판단되고 집기 조건을 만족한 물체만 집어 지정 수거함으로 이동한다.
6. 분실물 후보와 불확실한 물체는 건드리지 않고 결과에 기록하거나 운영자 검토 대상으로 남긴다.
7. 작업 완료 후 로봇은 대기 위치로 복귀한다.

### 4.2 시연 방식 후보

- P1: 사전에 정한 쓰레기 물체의 인식·분류·집기를 현장에서 시연한다.
- P2: 분실물/쓰레기 구분과 자율주행은 현장 안정성에 따라 시연하거나 영상으로 보완한다.
- P3: 초기 지도 생성, 공간 세팅, 대시보드 전체 흐름은 사전 준비 데이터 또는 영상으로 보완할 수 있다.

### 4.3 시나리오 참여자

| 참여자     | 역할                             |
| ------- | ------------------------------ |
| 이용자     | 공간을 사용하고 퇴실하는 사람               |
| 운영자     | 무인 공간의 품질과 상태를 관리하는 사람         |
| 끌리니 로봇  | 이용 후 정리·점검 작업을 수행하는 로봇         |
| 관제 대시보드 | 초기에는 운영자 호출과 상태 확인을 위한 후보 구성요소 |

### 4.4 작업 대상

| 구분                           | 1차 MVP 처리      |
| ---------------------------- | -------------- |
| 사전에 정한 쓰레기 물체                | 인식·분류·집기 대상    |
| 분실물 후보                       | 구분만 하고 건드리지 않음 |
| 의자·테이블 주변 정돈, 책상 표면, 조명, 출입문 | 후속 기능 후보       |
