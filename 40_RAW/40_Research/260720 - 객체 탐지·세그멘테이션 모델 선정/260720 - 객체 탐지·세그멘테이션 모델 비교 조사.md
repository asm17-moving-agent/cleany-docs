# 260720 - 객체 탐지·세그멘테이션 모델 비교

## 1. 목표 및 결론

Jetson Orin NX 16GB의 메모리와 연산 성능을 고려해 객체 탐지·세그멘테이션 모델을 비교했다. 현재 우선 선정안은 `YOLO11n-seg`다. 박스와 마스크를 함께 출력하면서 후보 중 규모가 가장 작고, 공개 정확도와 TensorRT 변환 경로가 확인된 점을 반영했다.

| 선정 단위        | 박스와 마스크를 함께 출력하는 단일 세그멘테이션 모델                              | 자료 기반 우선안, 팀 확인 필요                               |

## 3. 필요한 기능과 기술

## 4. Orin NX 16GB 메모리 예산

Jetson의 CPU와 GPU는 16GB 시스템 메모리를 공유하므로 모델 단독 사용량만으로 적합성을 판단하지 않는다. 실제 운영 작업을 실행한 상태에서 전체 최대 사용량과 남은 안전 여유를 함께 기록한다.

### 4.1 하드웨어 제약

| 항목 | Orin NX 16GB 기준 | 적용 내용 |
|---|---:|---|
| 통합 메모리 | 16GB | 운영체제, ROS 2, 카메라와 AI 모델이 함께 사용 |
| 메모리 대역폭 | 102.4GB/s | 여러 AI 모델을 동시에 실행하면 병목이 될 수 있음 |
| AI 연산 성능 | 25W에서 100 TOPS, 40W Super Mode에서 최대 157 TOPS | 전력 모드별 속도와 발열을 실제 장치에서 확인 |
| CUDA 코어 | 1,024개 | GPU를 다른 AI 작업과 나눠 써야 함 |
| DLA | 2코어, 614MHz | 지연시간 단축보다 GPU 사용량과 전력 부담을 줄이는 용도로 검토 |

### 4.3 메모리 판정 규칙

- 측정 도구는 `tegrastats`와 프로세스별 메모리(RSS)를 함께 사용하고 측정 주기를 기록한다.
- 대기 상태, TensorRT 엔진을 불러온 직후, 예열, 1,000회 연속 실행, 종료 직전의 최대값을 구분한다.
- ROS 2, RGB-D sensor, 주행, 조작과 실제로 함께 실행할 AI 작업을 포함한다.
- swap 사용, 메모리 부족(OOM) 또는 계속되는 메모리 압박이 발생한 후보는 배제한다.
- 안전 여유와 프로세스별 예산은 실측 후 팀 리뷰에서 확정한다.

| YOLOv8n | JetPack 6.0, TensorRT 10.0.1  | FP16 | 3.19ms |  약 313 | mAP50 0.52, mAP50-95 0.37 | 객체 탐지·추론만 측정         |
| YOLOv8n | JetPack 6.0, TensorRT 10.0.1  | INT8 | 2.32ms |  약 431 | mAP50 0.46, mAP50-95 0.32 | FP16보다 정확도 하락        |
| YOLO26n | JetPack 6, Ultralytics 8.4.33 | FP16 | 4.13ms |  약 242 | mAP50-95 0.4789           | COCO128 객체 탐지·추론만 측정 |
| YOLO26n | JetPack 6, Ultralytics 8.4.33 | INT8 | 3.49ms |  약 287 | mAP50-95 0.4489           | COCO128 기준, 정확도 하락   |

실제 끌리니 속도는 카메라 입력, 전처리, NMS와 마스크 후처리를 포함해 측정해야 한다.

### 6.1 제외 기준

| 기준 | 제외 조건 | 우선 기준 | 확인 상태 |
|---|---|---|---|
| 기능 지원 | 박스와 인스턴스 마스크를 함께 출력하지 못함 | 현재 단일 모델 선정 대상에서 제외 | 팀 확인 필요 |
| TensorRT 실행 | JetPack 6.2·TensorRT 10.3에서 FP16 엔진 생성·불러오기·추론에 실패함 | 실패하거나 허용되지 않은 별도 플러그인에 의존하면 제외 | 팀 확인 필요 |
| 메모리 | 동시 실행 조건의 최대 메모리가 예산이나 안전 여유를 침범함 | 모델 실행 메모리 1GiB 이하를 우선 검토하며 최종 한도는 팀이 결정 | 팀 확인 필요 |
| 속도 | 처리량이나 지연시간 기준에 미달함 | 전체 처리 속도 25 FPS 이상, p95 40ms 이하 | 팀 확인 필요 |
| 라이선스 | 프로젝트 이용·배포 조건을 수용할 수 없음 | 법적 의무를 수용할 수 없으면 제외 | 팀·라이선스 검토 필요 |
| 재현 가능성 | 배포판, 가중치 파일 또는 변환 경로를 재현할 수 없음 | 근거를 확보할 때까지 최종 비교 대상에서 보류 | 팀 확인 필요 |

## 8. 최종 선정 모델과 선정 근거

세 자료를 종합한 우선 선정안은 `YOLO11n-seg`다. Orin NX 실측과 팀 리뷰가 없으므로 아직 최종 선정으로 확정하지 않으며, 이 원본 조사 문서만으로 최종 결정이 되지 않는다.

| 항목             | 최종 내용                                                                                |
| -------------- | ------------------------------------------------------------------------------------ |
| 선정 상태          | 우선 선정안. 실측·팀 리뷰 후 최종 확정 필요                                                           |
| 선정 모델          | YOLO11n-seg                                                                          |
| 정확한 버전·가중치     | 배포판과 파일 확인값 고정 필요                                                                    |
| 담당 기능          | 객체 탐지 + 인스턴스 세그멘테이션                                                                  |
| 입력 크기·정밀도      | 640×640, batch 1, TensorRT FP16 기준                                                   |
| TensorRT 엔진 설정 | ONNX → TensorRT FP16. INT8·DLA는 JetPack 6.2·TensorRT 10.3에서 별도 검증                    |
| 공개 정확도         | COCO box mAP50-95 38.9, mask mAP50-95 32.0                                           |
| 끌리니 정확도        | 미측정. mAP50 ≥ 0.75 우선 목표                                                              |
| FPS·p50·p95    | Orin NX 세그멘테이션 실측 없음. 약 40~60 FPS는 입력 자료의 추정치                                        |
| 엔진 크기·실행 메모리   | 약 6~15MB, 약 0.3~1.2GB로 자료 간 추정치가 다르며 실측 필요                                           |
| 선정 근거          | 최종 비교 대상 중 가장 작은 2.9M 파라미터, 박스·마스크 동시 출력, 공개 마스크 정확도 사전 기준 충족, TensorRT 변환 경로 자료 존재  |
| 감수한 장단점        | YOLO11s보다 공개 마스크 mAP가 5.8 낮음, 작거나 얇은 물체의 성능 미검증, AGPL-3.0 표기 관련 이용·배포 검토 필요          |
| 재검토 조건         | 끌리니 정확도 목표 미달, 전체 처리 속도 25 FPS 미만, p95 40ms 초과, 최대 메모리 상한 초과, FP16 엔진 실패, 라이선스 수용 불가 |
