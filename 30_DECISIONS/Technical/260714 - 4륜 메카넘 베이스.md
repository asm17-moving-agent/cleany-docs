# 260714 - 4륜 메카넘 베이스

## 1. 결정

끌리니의 이동 베이스는 표준 XLeRobot의 3륜 omni 구성을 그대로 사용하지 않고, 4개의 Mecanum wheel과 독립 구동 모터를 사용하는 holonomic 커스텀 베이스로 구성한다.

상위 주행 계약은 `base_link` 기준 `linear.x`, `linear.y`, `angular.z`를 지원하도록 설계한다. DC motor encoder feedback과 MDD20A 출력은 멘토 지원 MCU가 담당한다. Mecanum wheel set과 4개 motor의 shaft, hub 및 체결 호환성은 확인했다. 정확한 MCU 모델과 통신 계약, wheel geometry, 속도·가속도 제한 및 안전 정지 회로는 후속 설계에서 확정한다.

## 2. 이유

- 4륜 Mecanum 구성은 전후 이동, 좌우 이동과 제자리 회전을 하나의 holonomic base 계약으로 표현할 수 있다.
- base kinematics와 `cmd_vel.linear.y` 지원 여부를 확정해야 simulation과 실제 로봇이 같은 속도 의미를 사용할 수 있다.
- encoder feedback과 motor 출력을 처리하는 controller는 멘토 지원 MCU를 사용한다.
- 표준 XLeRobot base와 다른 커스텀 구성이므로 BOM, robot description, odometry와 controller를 별도로 검증해야 한다.

## 3. 대안

| 대안                       | 선택하지 않은 이유                                 |
| ------------------------ | ------------------------------------------ |
| 표준 XLeRobot 3륜 omni base | 4륜 Mecanum 사용 확정 요청 및 현재 BOM 구성과 일치하지 않는다. |
| 차동구동 base                | 좌우 병진을 포함하는 holonomic 계약과 일치하지 않는다.        |
| 4륜 Mecanum custom base   | 현재 확정 요청과 BOM에 맞는 기준안이다.                   |

## 4. 가정

- 멘토 지원 MCU의 제공 및 적용 경로는 확인됐다. MCU 모델, encoder 입력 수와 전기 규격, control frequency, Jetson 통신 방식 및 firmware 책임은 추가 확인한다.
- 실제 wheel radius, wheelbase, track width, roller 방향과 motor 배치는 robot description과 controller config에서 같은 값으로 관리한다.
- 최종 payload와 중심 높이에서 motor torque, driver current와 전원 용량이 충분한지는 아직 검증되지 않았다.

## 5. 리스크

- custom Mecanum base용 assembly, URDF와 controller adaptation은 진행 중(WIP)이며, upstream 구성을 그대로 재사용할 수 없는 차이가 통합 일정에 영향을 줄 수 있다.
- wheel 배치, roller 방향 또는 kinematics matrix가 잘못되면 실제 이동 방향과 `cmd_vel` 의미가 달라질 수 있다. 상세 값과 검증 방법은 [[20_TECHNICAL/99 - Questions|Technical Questions]]에서 관리한다.
- Mecanum wheel slip으로 wheel odometry 오차가 커질 수 있다. encoder 기반 wheel odometry와 IMU를 융합하고 SLAM/localization의 절대 위치 보정으로 누적 오차를 제한하되, IMU만으로 평면 병진 slip을 제거할 수는 없다.
- MDD20A의 과전류·과열·저전압 보호는 활용한다. 다만 제조사 사양상 reverse-voltage protection은 제공되지 않으므로 mobile power, fuse, e-stop 또는 motor power cut-off, reverse-polarity protection은 별도로 정의하고 검증한다.

## 6. 재검토 조건

- 목표 payload에서 motor torque, 발열, 정지 거리 또는 배터리 지속 시간이 기준을 만족하지 못한다.
- 실내 바닥에서 slip과 odometry 오차가 Nav2 localization 허용 범위를 넘는다.
- frame, wheel mount 또는 chassis 구조가 4륜 Mecanum 배치를 지지하지 못한다.
- 멘토 지원 MCU가 4개 encoder 입력, 필요한 PWM/DIR 출력 또는 목표 control frequency를 지원하지 못한다.
- 예산이나 조달 제약으로 motor, wheel 또는 driver 구성을 변경해야 한다.
