# Rule-based VLA 아키텍처

## 1. 요약

Rule-based VLA는 물체의 의미와 주변 맥락을 해석해 행동 후보를 생성하고, 규칙 기반 검증을 거쳐 수거, 보관, 밀기, 정렬, 대기 중 하나의 행동으로 변환하는 예비 판단 구조다.

## 2. 기획 맥락

끌리니는 무인 공간의 쓰레기, 분실물 후보, 흐트러진 집기류를 구분하고 적절한 정리 행동을 선택해야 한다. 완전 자율 범용 판단보다는 기획서 기준 Rule-based VLA로 제한된 시나리오를 안정적으로 처리하는 방향이다.

## 3. 기술 개념

### 3.1 3 Layer 개념 초안

기획서에는 `3 Layer Rule-based VLA 작업 계획`이 언급되어 있다. 아래는 문서화를 위한 개념 초안이며, 실제 계층명과 책임은 검토 필요다.

| 계층 | 역할 | 상태 |
|---|---|---|
| Perception Layer | 물체 후보, 위치, 범주, 주변 맥락을 제공 | 기획서 기반 초안 |
| Semantic/Action Candidate Layer | VLA가 물체 의미와 주변 맥락을 해석해 행동 후보 생성 | 기획서 기반 초안 |
| Rule Validation/Execution Layer | 규칙 기반 검증 후 수거, 보관, 밀기, 정렬, 대기 행동으로 변환 | 기획서 기반 초안 |

### 3.2 판단 주기

- VLA 판단은 긴 주기로 행동을 명령한다.
- 실제 행동은 짧은 주기로 실행한다.
- 이 구조는 연산량을 최적화하고 체감 지연 시간을 줄이기 위한 것으로 기획서에 설명되어 있다.

### 3.3 행동 후보

- 수거: 쓰레기 후보를 지정 수거함으로 이동한다.
- 보관: 분실물 후보를 별도 보관 또는 알림 대상으로 처리한다. 구체 방식은 검토 필요다.
- 밀기: 위치가 어긋난 물체를 밀어 정렬한다.
- 정렬: 의자나 소형 집기를 기준 위치에 맞춘다.
- 대기: 불확실하거나 위험한 경우 행동하지 않는다.

## 4. 인터페이스 / 경계

| 구성요소 | 책임 | 경계 |
|---|---|---|
| Object Detection/Segmentation | 물체 후보와 영역 제공 | 행동 결정을 확정하지 않음 |
| 경량 VLM/VLA | 의미와 맥락 해석, 행동 후보 생성 | 안전 규칙을 우회하지 않음 |
| Rule Engine | 작업 가능 여부, 위험 여부, 분류 신뢰도 검증 | 저수준 모션 제어를 직접 구현하지 않음 |
| Action Executor | 선택된 행동을 로봇 동작으로 실행 | 행동 의미 재해석을 하지 않음 |
| Safety Monitor | 충돌, 사람, 조작 실패 등 위험 조건 확인 | 기획서에는 상세 기준이 없어 추가 정의 필요 |

## 5. 가정

- 물체 후보는 쓰레기, 분실물 후보, 기타 물체로 분류할 수 있다.
- 모든 행동은 규칙 기반 검증을 통과해야 실행된다.
- 불확실한 경우 대기하거나 운영자에게 알리는 정책이 필요하다.

## 6. 리스크

- 분실물과 쓰레기를 잘못 구분하면 사용자 자산 손상 문제가 발생할 수 있다.
- VLA 판단 근거가 불명확하면 디버깅과 평가가 어려워진다.
- 규칙이 과도하게 단순하면 실제 상황을 처리하지 못하고, 과도하게 복잡하면 유지보수가 어려워진다.

## 7. 관련 결정

- 현재 selected Decision 없음.
- Rule-based VLA 계층 구조 확정은 Technical Decision 후보다.

## 8. 미해결 질문

- 3 Layer의 공식 명칭과 책임은 무엇인가?
- VLA 후보 모델은 무엇이며 경량 VLM과의 관계는 무엇인가?
- 규칙 기반 검증의 최소 규칙 세트는 무엇인가?
- 불확실한 물체에 대한 기본 행동은 보관, 대기, 알림 중 무엇인가?
