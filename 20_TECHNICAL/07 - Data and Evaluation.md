# 데이터와 평가(Data and Evaluation)

## 1. 요약

끌리니의 데이터와 평가는 객체 탐지, Segmentation, 위치 추정, 시뮬레이션 기반 검증, Sim-to-Real 개선 구조를 중심으로 설계될 필요가 있다. 정량 평가 지표는 기획서에 상세하지 않아 추가 정의 필요다.

## 2. 기획 맥락

로봇이 실제 무인 공간에서 안정적으로 동작하려면 쓰레기, 분실물 후보, 집기류, 장애물을 인식하고 정리 행동의 성공 여부를 평가해야 한다. 실제 로봇 반복 실험이 어려운 주행, 접근, 파지, 수거, 정돈 시나리오는 MuJoCo와 Isaac Sim을 활용해 검증하는 방향이 기획서에 포함되어 있다.

## 3. 기술 개념

### 3.1 데이터 후보

- 책상 위 물체 이미지/깊이 데이터
- 바닥 쓰레기 후보 이미지/깊이 데이터
- 의자와 테이블 주변 정돈 상태 데이터
- 분실물 후보와 쓰레기 구분 사례
- 주행 중 장애물 데이터
- 로봇 작업 성공/실패 로그
- 시뮬레이션 생성 데이터

### 3.2 모델/기능 평가 후보

| 영역 | 평가 후보 | 상태 |
|---|---|---|
| 객체 탐지 | 쓰레기 후보 탐지 성공률 | 추가 정의 필요 |
| Segmentation | 물체 영역 분할 품질 | 추가 정의 필요 |
| 분류 | 쓰레기/분실물 후보/기타 물체 구분 정확도 | 추가 정의 필요 |
| 위치 추정 | 조작 가능한 물체 위치 추정 정확도 | 추가 정의 필요 |
| 주행 | 목적지 도달률, 장애물 회피 성공률 | 추가 정의 필요 |
| 조작 | 파지 성공률, 수거 성공률, 정렬 성공률 | 추가 정의 필요 |
| 통합 데모 | 이용 종료 후 정리 시나리오 완수 여부 | 추가 정의 필요 |

### 3.3 시뮬레이션 기반 검증

- MuJoCo와 Isaac Sim으로 주행, 접근, 파지, 수거, 정돈 시나리오를 가상 재현한다.
- 시뮬레이션에서 검증한 정책을 실제 XLeRobot 환경에 적용한다.
- 실제 실패 사례를 다시 시뮬레이션 조건에 반영하는 Sim-to-Real 개선 구조를 적용한다.

## 4. 인터페이스 / 경계

| 구성요소 | 책임 | 경계 |
|---|---|---|
| Data Collection | 이미지, 깊이, 로그, 실패 사례 수집 | 개인정보/보안 정책은 추가 정의 필요 |
| Dataset Curation | 쓰레기, 분실물 후보, 기타 물체 라벨 정리 | 라벨 기준 미정 |
| Simulation | 반복 실험과 정책 검증 | 실제 환경 성능을 완전히 보장하지 않음 |
| Evaluation | 지표 계산과 결과 비교 | 정량 목표값은 아직 미정 |
| Training Server | 로그 정제 및 학습 | 기획서에 세부 구조 없음 |

## 5. 가정

- 실제 로봇 실험 전 시뮬레이션 검증이 필요하다.
- 실패 사례는 다음 학습/검증 데이터에 반영된다.
- 평가 지표는 Planning 문서의 성공 기준과 연결되어야 한다.

## 6. 리스크

- 실제 공간 데이터가 부족하면 모델이 현장 상황을 일반화하지 못할 수 있다.
- 시뮬레이션과 실제 로봇 간 차이로 Sim-to-Real 성능 저하가 발생할 수 있다.
- 분실물 후보 라벨 기준이 불명확하면 오분류 리스크가 커진다.

## 7. 관련 결정

- 현재 selected Decision 없음.

## 8. 미해결 질문

- 최소 데이터셋 범위와 라벨 기준은 무엇인가?
- 평가 지표와 목표값은 어떻게 정할 것인가?
- 시뮬레이션으로 검증할 시나리오와 실제 테스트로 검증할 시나리오는 어떻게 나눌 것인가?
- 현장 데이터 수집 시 개인정보 또는 보안 이슈가 있는가?
