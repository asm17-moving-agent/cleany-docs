# 안전과 리스크(Safety and Risk)

## 1. 요약

끌리니의 주요 안전 리스크는 장애물 회피, 사람 없는 시간대 운영, 문단속과 소등, 조작 실패, 물체 오분류, 파지 실패, 충돌 위험이다. 기획서에는 안전 기준이 상세하지 않으므로 추가 정의 필요다.

## 2. 기획 맥락

무인 공간에서 로봇이 이동하고 물체를 조작하면 시설물, 사용자 소유물, 로봇 자체에 대한 위험이 생길 수 있다. 특히 쓰레기와 분실물 후보를 잘못 판단하거나, 문단속/소등 같은 시설 조작을 잘못 수행하면 운영 리스크가 커진다.

## 3. 기술 개념

### 3.1 리스크 유형

| 리스크 | 설명 | 현재 대응 상태 |
|---|---|---|
| 장애물 회피 실패 | 의자, 가방, 사람, 임시 장애물과 충돌 가능 | 추가 정의 필요 |
| 사람 감지 실패 | 사람 없는 시간대 운영을 전제로 해도 예외 상황 가능 | 추가 정의 필요 |
| 물체 오분류 | 분실물을 쓰레기로 오인하거나 반대 상황 발생 | 추가 정의 필요 |
| 파지 실패 | 물체를 떨어뜨리거나 손상시킬 가능성 | 추가 정의 필요 |
| 조작 실패 | 의자 정렬, 책상 닦기, 문단속 등 실패 가능 | 추가 정의 필요 |
| 소등/문단속 오류 | 시설 상태를 잘못 변경할 가능성 | 추가 확인 필요 |
| 데이터/프라이버시 | 카메라 기반 데이터 수집 시 개인정보 가능성 | 추가 확인 필요 |
| 온디바이스 장애 | 추론 지연, 노드 중단, 발열, 전력 문제 | 추가 정의 필요 |

### 3.2 기본 안전 원칙 초안

- 불확실한 물체는 쓰레기로 단정하지 않는다.
- 사람 또는 예기치 않은 장애물이 감지되면 보수적으로 정지하거나 대기한다.
- 문단속과 소등은 실제 시설 인터페이스와 운영 정책이 확인되기 전까지 placeholder로 취급한다.
- 실패는 숨기지 않고 로그 또는 대시보드에 남기는 방향으로 검토한다.

## 4. 인터페이스 / 경계

| 구성요소 | 책임 | 경계 |
|---|---|---|
| Perception | 사람, 장애물, 물체 후보 감지 | 최종 안전 판정을 단독으로 확정하지 않음 |
| Rule Engine | 위험 조건에서 행동 차단 | 법적/시설 정책을 대체하지 않음 |
| Navigation Safety | 이동 중 장애물 회피와 정지 | 조작 중 충돌 방지는 별도 고려 필요 |
| Manipulation Safety | 집기, 밀기, 닦기 중 힘/위치 제한 | 구체 제어 기준 미정 |
| Dashboard/Log | 실패와 위험 이벤트 기록 | 즉각 대응 체계는 추가 정의 필요 |

## 5. 가정

- 초기 운영은 사람 없는 시간대 또는 사람이 적은 시간대를 우선 고려한다.
- 안전 기준은 기획서에 상세하지 않아 별도 정의가 필요하다.
- 로봇은 불확실한 상황에서 보수적으로 행동해야 한다.

## 6. 리스크

- 안전 기준 부재는 데모와 상용화 가능성 모두에 영향을 준다.
- 오분류는 사용자 자산 훼손 또는 고객 불만으로 이어질 수 있다.
- 문단속/소등은 시설별 인터페이스 차이가 크다.

## 7. 관련 결정

- 현재 selected Decision 없음.
- 안전 기준과 실패 처리 정책은 Technical Decision 후보다.
